Robotic Soldering Arm

An active helping hand for precise PCB soldering and manipulation. Project by Dylan Huber.

Introduction

Soldering components onto a Printed Circuit Board (PCB) can be a tricky task, often feeling like you need an extra pair of hands. One hand holds the soldering iron, another the solder, and somehow you still need to keep the PCB steady and correctly positioned. This project introduces a robotic arm designed to be that much-needed "third hand." It securely grips a PCB, allowing for easy manipulation and stable positioning, ultimately making the soldering process more efficient and precise. The arm features different operational modes for movement and the ability to save and recall specific positions.

The Problem: The "Not Enough Hands" Dilemma

The core challenge in manual soldering is managing multiple tools and the workpiece simultaneously:

  • Holding the soldering iron.
  • Feeding the solder wire.
  • Keeping the PCB stable and at the correct angle.

This juggling act often leads to frustration, slower work, and potentially lower quality solder joints. While simple holding tools like flexible metal arms exist, they can have drawbacks:

  • They may not be strong enough to resist movement during soldering.
  • Maneuvering them into the perfect position can be cumbersome.
  • Returning to previous positions is tedious.

Supplies / Bill of Materials

To build this Robotic Soldering Assistant, you'll need the following:

Hardware:

  • Robotic Arm: An articulated robotic arm with a gripper compatible with the xarm library.
  • Control Unit: A computer to run the Python control scripts. This could be:
    • A single-board computer like a Raspberry Pi (ensure it has enough processing power and USB ports).
    • A laptop or desktop computer.
  • USB Cable: To connect the robotic arm to the control unit for communication.
  • Power Supply: A power supply unit (PSU) appropriate for the specific robotic arm model, capable of delivering sufficient voltage and current for all servos.
  • (Optional) PCB & Components: The items you intend to solder.
  • (Optional) Soldering Iron, Solder, Flux: Standard soldering equipment.

Software & Libraries:

  • Python: Version 3.x.
  • xarm Library: Python library for controlling xArm robotic arms. Install via pip: pip install xarm.
  • pynput Library: Python library for capturing keyboard input. Install via pip: pip install pynput.
  • pickle Library: Part of the Python Standard Library (no separate installation needed), used for saving and loading arm states.

The Solution: A Smart Robotic Helping Hand

This project tackles the "not enough hands" problem by employing a robotic arm to securely hold and position the PCB. This provides a stable and maneuverable platform for soldering tasks.

Key Features:

  • Dual Operational Modes:
    • Mode 1 (Power Steering/Assisted Movement): Enables smooth, servo-assisted movement of the arm, allowing for quick and intuitive repositioning of the PCB.
    • Mode 2 (Stationary): Locks the arm firmly in place, providing a stable platform that resists external forces during the soldering process.
    • The user can easily toggle between these modes using keyboard commands.
  • Programmable Setpoints (Saved Positions):
    • Users can save specific arm orientations (positions of all servos).
    • These saved positions can be instantly recalled, facilitating quick transitions between preferred working angles or when moving between different sections of a PCB.
    • Saved orientations are stored persistently and can be loaded across different sessions.
  • Adjustable Grip: The arm's claw can be opened and closed via keyboard command to securely hold various sizes of PCBs.

How It Works

The robotic arm is driven by a Python application that interfaces with its servo motors. The system is designed to interpret user commands for movement, mode switching, and position management.

Servo Control:

The xarm Python library is employed for low-level control of these servos. The main script, arm_controller.py, is responsible for initializing the arm and managing servo communications.

State Management:

The ArmState.py script defines a class that represents the collective state of the arm, essentially a snapshot of all servo positions. The Arm.py script leverages this class to get and set the arm's configuration, ensuring coordinated movements.

User Input:

User interaction is handled via keyboard input, facilitated by the pynput library within arm_controller.py. This allows the user to:

  • Toggle the claw open or closed (Space bar).
  • Manually input target positions for individual servos (e.g., 1.500 then Enter would set the first servo to position 500, all servos range from position 0 to 1000).
  • Save the current arm position to a setpoint (ex: s1 then Enter to save to state 1, g1 then Enter to go to state 1) or move to a previously saved setpoint.
  • Switch between "Assisted Movement" and "Stationary" modes (a key).
  • Turn off power to the servos (Tab key).

Position Averaging and Velocity (Assisted Mode):

The Arm.py script implements the assisted mode by finding the arm velocity and then matching it to help users get to the position they are targeting.

Saving/Loading States:

A key feature is the ability to save and load arm configurations, even between sessions. The Arm.py script uses Python's pickle library to serialize ArmState objects to a file (saved_states.pkl). This allows users to preserve preferred positions between uses.

Challenges Encountered

  • Maintaining Stationary Position: Achieving perfect stillness in the arm was difficult. Factors like gravity constantly pulling on the arm segments, backlash in servo gears, and servos that don't like to stall contributed to this challenge.
  • Assisted Mode Implementation: Getting the "power steering" or assisted movement mode to feel natural took a lot of fine tuning and testing.

Next Steps & Future Improvements

  • Make a better arm model:
    • Use positions to calculate exact forces upon the arm. Find a library that allows direct torque setting and counter these forces.
    • Use this to make a better stationary mode where the robot is applying the perfect torques to prevent falling.
    • Use this to make a better assisted mode where the robot is applying the correct torques to make the joints feel frictionless and the arm to feel weightless when forcing movement.
  • Develop a Graphical User Interface (GUI):
    • Transition from keyboard commands to a more user-friendly GUI (e.g., using PyQt, Kivy, or a web interface with Eel/Flask).
    • Provide visual feedback of the arm's current position and servo states.
    • Allow easier management of saved positions with naming and visual selection.
    • Provide clearer feedback to the user for invalid commands or operational issues.
  • Safety Features:
    • Implement software-defined "forbidden zones" or more precise joint limit checks beyond basic clamping (including no self-collisions or going below the table it is sitting upon unless specified to).
    • Ensure that the arm stops if the forces it is applying are being stopped.

Conclusion

The Robotic Soldering Arm project successfully created a better version of the "helping hands" currently used in the soldering world. It has key features like programmable setpoints, assisted movement, and strong resistance to movement while soldering. The project is a step towards making soldering easier and more efficient, and it opens the door for future improvements and enhancements. Additionally, this platform can be adapted for other applications, with minor changes to the code base (really just holding anything in the claw can be helpful when working on a physical project).
